## 文档结构

driver-box的文档使用Starlight主题构建，主要分为以下几个部分：

- **快速开始** (`guides/`) - 帮助用户快速上手使用driver-box
- **概念介绍** (`concepts/`) - 解释核心概念和技术原理
- **通信插件** (`plugins/`) - 各种通信插件的详细说明
- **Export** (`export/`) - Export 功能模块的使用说明

## 写作风格指南

### 语言风格

- 使用简洁、明确的语言
- 保持一致的语气，以第二人称（你）与读者对话
- 避免使用过于技术化的术语，如必须使用，请先定义清楚
- 保持积极正面的语调

### 标题和层级

- 使用Markdown标题层级（#、##、###）
- 标题应简洁明了，准确反映内容
- 每个页面标题使用一级标题（#），章节使用二级标题（##）

### 示例代码

- 所有代码示例都需要有适当的语法高亮
- 提供完整的、可运行的示例
- 在复杂示例中添加必要的注释
- 使用实际的场景和有意义的变量名

## 技术规范

### Markdown格式

driver-box文档使用标准Markdown格式，并支持以下扩展：

```markdown
---
title: 文档标题
sidebar:
  order: 序号
---

# 主标题

这是段落内容。

## 子标题

### 三级标题

- 列表项1
- 列表项2

`内联代码`

```代码块
```


### 特殊组件

文档支持Starlight主题提供的特殊组件：

#### 侧边栏标签

```markdown
---
title: 文档标题
sidebar:
  order: 1
---
```

#### 通知框（Asides）

```mdxjs

</Aside>

<Aside type="caution" title="警告">
这是一个警告框
</Aside>
```


## 内容质量要求

### 完整性

- 确保文档包含所有必要信息
- 每个概念都应该有清晰的解释
- 提供足够的上下文信息
- 包含常见问题的解决方案

### 准确性

- 确保所有技术信息准确无误
- 代码示例经过验证，可以正常运行
- 链接有效且指向正确位置
- 避免过时的信息

### 易读性

- 将长段落分解为更小的部分
- 使用列表和表格组织信息
- 提供视觉层次结构
- 保持逻辑连贯

## 文档类型及写作要点

### 快速入门指南

- 提供清晰的逐步说明
- 最小化初始设置步骤
- 包含常见错误的解决方法
- 提供立即可见的结果

### API参考文档

- 准确定义参数类型和返回值
- 说明每个参数的作用
- 提供使用示例
- 标明废弃或实验性功能

### 概念性文档

- 解释为什么使用某项功能
- 提供背景知识
- 使用类比帮助理解
- 避免过多的技术细节

### 教程文档

- 有明确的学习目标
- 步骤按逻辑顺序排列
- 提供预期结果
- 包含故障排除信息


## 验收标准

您的文档贡献需要满足以下标准：
- 没有语法和拼写错误
- 符合本指南的格式要求
- 内容准确且最新
- 示例代码可运行
- 链接有效
- 适合目标读者群体

### 检查清单

在提交前，请检查：

- [ ] 标题是否清晰准确
- [ ] 内容是否完整
- [ ] 是否有语法和拼写错误
- [ ] 代码示例是否有效
- [ ] 链接是否可用
- [ ] 图片是否加载正常
- [ ] 是否遵循样式指南
